## 7. RAG (Retrieval-Augmented Generation) — The Key Skill (Weeks 8–10)

> **This is the MOST important section.** RAG is the #1 skill companies hire for.

### Videos

| # | Resource | Channel | Duration | Link |
|---|----------|---------|----------|------|
| 1 | RAG from Scratch (Full Playlist – 14 parts) | LangChain | 14 videos | [Watch](https://www.youtube.com/playlist?list=PLfaIDFEXuae2LXbO1_PKyVJiQ23ZztA0x) |
| 2 | RAG + LangChain Full Tutorial | freeCodeCamp | 2 hrs | [Watch](https://www.youtube.com/watch?v=tcqEUSNCn8I) |
| 3 | Build a RAG App – Step by Step | pixegami | 30 min | [Watch](https://www.youtube.com/watch?v=2TJxpyO3ei4) |
| 4 | Advanced RAG Techniques | Sam Witteveen | 25 min | [Watch](https://www.youtube.com/watch?v=TRjq7t2Ms5I) |
| 5 | Chat with Any PDF using RAG | Nicholas Renotte | 1 hr | [Watch](https://www.youtube.com/watch?v=dXxQ0LR-3Hg) |
| 6 | Production RAG – Best Practices | AI Makerspace | 1.5 hrs | [Watch](https://www.youtube.com/watch?v=f1gMfbRbo80) |

### What to Learn

#### Basic RAG Pipeline
```
Load Documents → Split into Chunks → Create Embeddings → Store in Vector DB → Retrieve Relevant Chunks → Generate Answer with LLM
```

#### Core Concepts
- **Document Loaders** — PDF, web scraping, CSV, Notion, GitHub, YouTube
- **Text Splitting** — RecursiveCharacterTextSplitter, semantic splitting
- **Chunk Size & Overlap** — How size affects retrieval quality
- **Embedding Models** — OpenAI `text-embedding-3-small`, Sentence Transformers
- **Vector Store** — Store and retrieve embeddings
- **Retrieval** — Similarity search, MMR (Maximal Marginal Relevance)
- **Generation** — Pass retrieved context + question to LLM

#### Advanced RAG Techniques
- **HyDE** (Hypothetical Document Embeddings) — Generate hypothetical answer, embed that for search
- **Multi-Query RAG** — Generate multiple search queries from one question
- **Parent-Child Chunks** — Store small chunks, retrieve parent documents
- **Re-ranking** — Use a cross-encoder to re-rank retrieved results
- **Hybrid Search** — Combine keyword search (BM25) + vector search
- **Self-RAG** — LLM decides when to retrieve and validates its own answers
- **Corrective RAG** — Check retrieved docs for relevance, fall back to web search

#### Evaluation
- **RAGAS Framework** — Faithfulness, answer relevance, context precision, context recall
- **LangSmith** — Trace and debug RAG pipelines
